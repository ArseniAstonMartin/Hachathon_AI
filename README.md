# FastAPIAuthService

Собственная система аутентификации и авторизации на базе **FastAPI**, **PostgreSQL** и **Redis**. Проект реализует кастомную логику разграничения прав доступа и управления сессиями, согласно требованиям тестового задания.

---

## Схема управления доступом (RBAC)
В соответствии с требованиями ТЗ, реализована следующая архитектура безопасности:

* **Идентификация**: Пользователь идентифицируется по JWT-токену (RS256) при каждом запросе.
* **Хранение паролей**: Применяется хеширование `bcrypt` с индивидуальной солью на основе email.
* **Управление сессиями**: Активные сессии хранятся в **Redis**, что позволяет мгновенно аннулировать доступ (Logout) или блокировать пользователей.
* **Роли**:
    * **User**: Доступ к своему профилю, обновление данных и мягкое удаление.
    * **Admin**: Полный доступ к защищенным ресурсам (проверка через кастомные зависимости FastAPI).
* **Обработка ошибок**:
    * **401 Unauthorized**: Выдается, если не удалось определить пользователя.
    * **403 Forbidden**: Выдается, если у пользователя недостаточно прав для ресурса.

##  Локальная установка

### 1. Создать файл .env в корне проекта
Скопируйте содержимое из примера и настройте параметры под себя:

```sh
cat .env.example > .env
```

### 2. Сгенерировать приватный ключ
Создайте приватный ключ для подписи JWT-токенов (алгоритм RS256):

```sh
openssl genpkey -algorithm RSA -out private.pem -pkeyopt rsa_keygen_bits:2048
```

### 3. Сгенерировать публичный ключ
Извлеките публичный ключ из созданного приватного для проверки подлинности токенов:

```sh
openssl rsa -in private.pem -pubout -out public.pem
```

### 4. Запустить проект через Docker
Автоматическая сборка и запуск приложения (FastAPI), базы данных (PostgreSQL) и кэша (Redis):

```sh
docker-compose up --build
```

## Тестирование и работа с БД
Запустить автоматические тесты
Полная проверка функционала (регистрация, логин, профиль) внутри контейнера:

```sh
docker-compose exec app pytest tests/auth/
```
Проверить данные в базе (psql)
Прямое подключение к PostgreSQL для просмотра пользователей и их статусов:

```sh
docker-compose exec db psql -U postgres -d AuthManager -c "SELECT worker_id, email, is_admin, is_active FROM worker_base;"
```

##  Реализованный функционал
* **Регистрация**: Ввод ФИО, email, пароля и обязательный повтор пароля.

* **Login:** Вход по email, выдача Access/Refresh токенов и запись сессии в Redis.

* **Logout:** Выход из системы с мгновенным удалением сессии из кэша.

* **Обновление информации:** Редактирование полей профиля через PATCH /me.

* **Удаление (мягкое):** Пользователь инициирует удаление, происходит logout, запись в БД получает статус is_active=False.
